Basic Data Structure Terminologies (Alphabetical Order)

  • Abstract Data Type (ADT):
    A type for objects whose behavior is defined by a set of values and operations, without specifying how those operations are implemented.
  • Algorithm: A step-by-step procedure or formula for solving a problem or performing a task.
  • Array: A linear data structure consisting of elements stored in contiguous memory locations, where each element can be accessed by its index.
  • Backtracking: An algorithmic technique for incrementally building a solution and abandoning those that fail to meet constraints.
  • Binary Search Tree (BST): A binary tree where each node's left child is less than the parent and the right child is greater.
  • Bottom: The first element added to a stack, and the last one to be removed.
  • Capacity: The maximum number of elements a data structure can hold.
  • Collision: When two different keys hash to the same index in a hash table.
  • Data Structure: A way of organizing and storing data to access and modify it efficiently.
  • Dequeue: Removing an element from the front of a queue.
  • Divide and Conquer: A technique for solving problems by dividing them into smaller subproblems, solving them recursively, and combining the results.
  • Doubly Linked List: A type of linked list where each node points to both the next and the previous node.
  • Dynamic Programming: A method of solving problems by breaking them down into overlapping subproblems and storing their solutions for reuse.
  • Element: A single data item stored in a data structure.
  • Empty: A data structure containing no elements.
  • Enqueue: Adding an element to the rear of a queue.
  • Field: A single piece of information stored in a data structure.
  • FIFO (First In, First Out): A principle where the first element added is the first one removed, used in queues.
  • Full: A data structure that can't hold any more elements.
  • Graph: A non-linear data structure with nodes (vertices) and edges (connections), which can be directed, undirected, or weighted.
  • Greedy Algorithm: An algorithm that makes the locally optimal choice at each step to try to find the global optimum.
  • Hash Table: A data structure that maps keys to values using a hash function to compute an index.
  • Hashing: A technique to map data to unique values (hash codes) using a hash function, mainly used for fast retrieval.
  • Heap: A special tree-based structure that satisfies the heap property, such as max-heap or min-heap.
  • Head: The first element in a linked list.
  • LIFO (Last In, First Out): A principle where the last element added is the first one removed, used in stacks.
  • Length: The total number of elements in a data structure.
  • Linked List: A linear data structure where each element (node) contains a reference to the next element.
  • Node: A basic unit of a data structure containing data and pointers to other nodes.
  • Null: A special value that indicates a pointer does not reference anything.
  • Overflow: An error that occurs when trying to add elements to a full data structure.
  • Pointer: A reference to another location in memory, often used in linked lists and trees.
  • Pop: Removing the top element from a stack.
  • Priority Queue: A data structure where elements are dequeued based on their priority.
  • Push: Adding an element to the top of a stack.
  • Queue: A linear data structure following FIFO, where elements are added to the rear and removed from the front.
  • Recursion: A function calling itself to solve smaller instances of the same problem.
  • Rear (or Back): The last element in a queue, where new elements are added.
  • Searching: The process of finding an element in a data structure.
  • Sorting: Arranging data in a specific order, such as ascending or descending.
  • Stack: A linear data structure that follows LIFO, where the last element added is the first one removed.
  • Tail: The last element in a linked list, often pointing to `null`.
  • Top: The last element added to a stack, which is accessed or removed first.
  • Traversal: The process of visiting all nodes or elements in a data structure.
  • Tree: A hierarchical data structure consisting of nodes with parent-child relationships.
  • Underflow: An error that occurs when trying to remove elements from an empty data structure.
